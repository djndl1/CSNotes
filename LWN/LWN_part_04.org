** Part 04 (Articles 2001-2500)

- [[https://lwn.net/Articles/683504/][Tracepoints with BPF]]: Alexei Starovoitov's patches for Linux 4.7 add BPF_PROG_TYPE_TRACEPOINT allowing BPF programs to attach to kernel tracepoints for in-kernel trace data processing (e.g. histogram bucketing), more efficient than kprobe-based approaches; raises ABI stability concerns since tracepoints expose kernel internals.
- [[https://lwn.net/Articles/682582/][Toward less-annoying background writeback]]: Jens Axboe's patches address block layer bufferbloat by throttling writeback requests when queues exceed per-device limits; tracks whether request is for memory reclaim or regular writeback to avoid OOM; implements 10ms delay for cached drives but aims for auto-tuning.
- [[https://lwn.net/Articles/683476/][Static code checks for the kernel]]: Arnd Bergmann's ELC 2016 talk covers his automated randconfig build testing that produced 500+ patches fixing compiler warnings; catches missing dependencies, uninitialized variables; compares Sparse, checkpatch, Smatch, Coverity, and Clang static analysis tools.
- [[https://lwn.net/Articles/683320/][OpenBMC, a distribution for baseboard management controllers]]: Facebook's open source BMC distribution built with Yocto replaces proprietary IPMI firmware; runs on ASPEED SoCs in Open Compute Project servers; exposes hardware monitoring via lm-sensors and SSH; IBM also developing OpenBMC for OpenPOWER platforms with upstream kernel submissions planned.
- [[https://lwn.net/Articles/683350/][Python looks at paths]]: Python core developers debated making pathlib.Path integrate with standard library; agreed on __fspath__() protocol and os.fspath() helper for explicit path-to-string conversion rather than making Path subclass str; Brett Cannon explains why Path doesn't inherit from str (not all strings are valid paths).
- [[https://lwn.net/Articles/682923/][Hutterer: Why libinput doesn't have a lot of config options]]: Peter Hutterer explains why libinput avoids configuration options - maintainers bear long-term cost of supporting each option including test cases, documentation, and bug reports; hardware quirks are handled via udev hwdb or remain private to libinput.
- [[https://lwn.net/Articles/682594/][The Internet of bricks]]: Revolv home automation hub bricking after Nest acquisition highlights IoT device dependency on cloud services; despite running Ångström Linux, users cannot update firmware since it's locked down; raises concerns about proprietary software controlling owned hardware.

- [[https://lwn.net/Articles/682591/][Redox: a Rust-based microkernel]]: Redox OS implements microkernel design with 16K LOC kernel in Rust for memory safety; uses "everything is a URL" scheme for resource access; MIT licensed; ZFS filesystem planned; does not intend to replace Linux.
- [[https://lwn.net/Articles/682302/][The LoadPin security module]]: Kees Cook's LoadPin module (used in ChromeOS since 2011) ensures all kernel file loads come from the same filesystem as the first load; prevents loading modules/firmware from untrusted filesystems on verified-boot systems.

- [[https://lwn.net/Articles/682391/][Improvements in CPU frequency management]]: Rafael Wysocki's schedutil cpufreq governor integrates with scheduler load tracking; removes cpufreq timer dependency; adds fast_switch() interface for interrupt-context frequency changes; aims for scheduler-driven power management.
- [[https://lwn.net/Articles/682538/][Early packet drop with BPF]]: Brenden Blanco's XDP (Express Data Path) patchset adds BPF_PROG_TYPE_PHYS_DEV for early packet processing in network drivers; aims for sub-SKB overhead packet handling before kernel networking stack.
- [[https://lwn.net/Articles/682721/][Python and to_file()]]: Python-ideas discussion on string-to-file convenience functions revealed complexity around atomic writes, buffered I/O, and cross-platform issues; pathlib's write_text() and read_text() available since Python 3.5; io module suggested as potential home.
- [[https://lwn.net/Articles/682051/][Mono Relicensed MIT]]: Xamarin (acquired by Microsoft) relicenses Mono runtime from LGPL dual-license to MIT; enables C#/.NET use in embedded apps and game engines without proprietary licensing concerns.
